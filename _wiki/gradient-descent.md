# Gradient Descent — 梯度下降

## 概述

**梯度下降（Gradient Descent）** 是深度學習中最核心的最佳化演算法。它透過反覆沿著損失函數對參數的梯度方向（最陡下降方向）更新參數，逐步逼近損失函數的最小值。所有神經網路的訓練本質上都是梯度下降或其變體的應用。mini-llm 專案使用 AdamW（一種梯度下降的進階變體）來訓練模型。

## 動機：從函數到最佳化

深度學習的本質是：**找到一組參數 θ 使得損失函數 L(θ) 最小化**。

```
θ* = argmin_θ L(θ)
```

對於線性迴歸，我們可以解析求解（正規方程式）。但對於多層神經網路，損失函數是高度非凸的，不存在解析解。我們需要**迭代方法**：從隨機初始點出發，逐步走向低損失區域。

## 基本原理

### 梯度：最快上升方向

一個多變數函數 `f(x₁, x₂, ..., xₙ)` 在點 `x` 的梯度是一個向量：

```
∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ
```

梯度的兩個關鍵性質：
1. **方向性** — 梯度指向函數在該點**增加最快**的方向
2. **大小** — 梯度的範數 `||∇f(x)||` 表示變化的劇烈程度

### 梯度下降更新規則

要讓函數值下降，我們往梯度的**反方向**移動：

```
x_{t+1} = x_t - η · ∇f(x_t)
```

其中：
- `x_t` — 第 t 步的參數值
- `∇f(x_t)` — 在 x_t 處的梯度
- `η` — 學習率（步長的超參數）

### 類比：登山者下山

想像一個在山谷中迷霧中的登山者想要下山：
- **梯度** = 登山者腳下最陡的方向（測量）
- **負梯度方向** = 往山下走的方向
- **學習率** = 每一步的大小
  - 太大：可能跨過山谷到另一側的山坡上
  - 太小：永遠走不到山底
- **收斂** = 到達谷底（損失最小值）

## 三種梯度下降的變體

### 1. Batch Gradient Descent（批次梯度下降）

每次更新使用**所有**訓練樣本計算梯度：

```
θ_{t+1} = θ_t - η · (1/N) · Σᵢⁿ ∇Lᵢ(θ_t)
```

| 優點 | 缺點 |
|------|------|
| 梯度方向準確 | 每次更新計算量大 |
| 收斂穩定 | 無法處理超過記憶體大小的資料集 |
| 理論分析方便 | 無法利用 GPU 的批次處理優勢 |

### 2. Stochastic Gradient Descent（隨機梯度下降）

每次更新只使用**一個**隨機樣本：

```
θ_{t+1} = θ_t - η · ∇Lᵢ(θ_t)
```

| 優點 | 缺點 |
|------|------|
| 每次更新非常快 | 梯度方向雜訊大 |
| 可以逃離局部極小值（雜訊的正面效果） | 收斂路徑曲折 |
| 可以處理流式／線上資料 | 需要更小的學習率來補償雜訊 |

### 3. Mini-batch Gradient Descent（小批次梯度下降）— 最常用

每次更新使用一個**小批次**（通常 16–512）樣本：

```
θ_{t+1} = θ_t - η · (1/B) · Σᵢᴮ ∇Lᵢ(θ_t)
```

這是 mini-llm 使用的方式（`batch_size = 32`），也是深度學習的標準做法。

## 學習率（Learning Rate）

學習率是最重要的超參數。它的選擇直接決定訓練的成敗。

### 學習率過高 vs 過低

```
學習率過低：               學習率過高：
Loss                       Loss
  | ↘                        | ↙↗
  |  ↘                       | ↙   ↗
  |   ↘                      | ↙     ↗
  |    ↘                     | ↙
  |     →                    | → time
  time                      
  (太慢收斂)                 (發散)
```

### 學習率排程（Learning Rate Schedule）

在訓練過程中動態調整學習率：

- **Step Decay** — 每固定步數乘以一個因子（如每 500 步 × 0.1）
- **Cosine Annealing** — 按照餘弦曲線從初始值下降到接近 0
- **Warmup** — 先從很小的學習率線性增長到目標值，再開始下降
- **Reduce on Plateau** — 當 loss 不再下降時降低學習率

mini-llm 使用固定學習率（最簡形式），但大規模訓練中學習率排程至關重要。

## 梯度消失與梯度爆炸

### 梯度消失（Vanishing Gradient）

在深層網路中，梯度可能隨著向輸入端傳播而指數級減少：

```
如果權重矩陣的最大奇異值 < 1：
    每層梯度 ≈ 原始梯度 × λ^層數
    λ < 1 → λ^10 ≈ 0 → 淺層參數幾乎不更新
```

**影響**：深層網路的淺層幾乎無法訓練。

**解決方案**：
- 使用 ReLU/Swish 等非飽和活化函數
- 殘差連接（ResNet / Transformer）
- 適當的權重初始化（Xavier / He / Kaiming）
- Layer / RMS 正規化

### 梯度爆炸（Exploding Gradient）

相反情況：梯度指數級增長，導致參數更新過大，loss 變為 NaN。

**解決方案**：
- 梯度裁切（Gradient Clipping）— mini-llm 使用 `max_norm=1.0`
- 權重正則化
- 較小的學習率

## 一階方法 vs 二階方法

| 特性 | 一階方法 (SGD/Adam) | 二階方法 (Newton/Quasi-Newton) |
|------|---------------------|-------------------------------|
| 使用資訊 | 梯度（一階導數） | 梯度 + Hessian（二階導數） |
| 每次迭代成本 | O(n) | O(n²) 或更高 |
| 收斂速度 | 線性 | 二次（更快） |
| 適合大模型 | 是 | 否（計算量太高） |
| 對曲率的適應 | 透過自適應方法間接處理 | 直接處理 |

深度學習中一階方法佔主導地位，因為參數量太大而無法計算 Hessian 矩陣。AdamW 作為自適應一階方法，在收斂速度和計算成本之間取得了良好的平衡。

## Backpropagation（反向傳播）

梯度下降依賴於高效的梯度計算，而反向傳播提供了這種計算方式。

### 鏈式法則（Chain Rule）

```
如果 z = f(y), y = g(x)
則 dz/dx = dz/dy · dy/dx
```

### 反向傳播的過程

```
前向傳播：x → h₁ → h₂ → ... → loss（計算每個節點的值）
反向傳播：loss → ... → ∂h₂/∂h₁ → ∂h₁/∂x（計算每個節點的梯度）
```

深度學習框架（PyTorch、TensorFlow）透過**計算圖**自動進行反向傳播：

```python
loss.backward()  # PyTorch 自動計算所有參數的梯度
```

在這個步驟之後，每個 `nn.Parameter` 的 `.grad` 屬性包含了對應的梯度。

## 隨機梯度的統計性質

雖然每個 mini-batch 的梯度 `g_B(θ)` 只是真實梯度 `∇L(θ)` 的**有雜訊估計**，但：

```
E[g_B(θ)] = ∇L(θ)  ← 無偏估計（unbiased）
Var(g_B(θ)) ∝ σ²/B ← 批次越大，估計越準確
```

這種雜訊實際上對訓練有益：
1. **逃離鞍點** — 梯度雜訊可以幫助模型逃離梯度接近 0 的鞍點
2. **隱式正則化** — 雜訊相當於在損失函數上添加了一個隱式的正則項

## 收斂理論

凸最佳化中，梯度下降有成熟的收斂理論。但神經網路的損失函數是高度非凸的，理論分析更困難。

### 非凸最佳化的挑戰

1. **局部極小值** — 非凸函數有多個局部極小值，不一定找到全局最小值
2. **鞍點** — 在高維空間中，鞍點比局部極小值更常見
3. **Plateau（平台）** — 大面積的平坦區域，梯度接近 0，幾乎不更新

### 經驗觀察

大量的經驗研究發現：
- 神經網路的局部極小值通常質量相似（loss 值接近）
- 更深的網路反而更容易找到好的極小值（深度學習的「反直覺」性質）
- 超參數的選擇對收斂品質的影響遠大於初始值

## 實例：追蹤 f(x)=x² 的梯度下降

考慮最簡單的單變數二次函數：

```
f(x) = x²,   f'(x) = 2x
```

最小值在 x=0, f(0)=0。從 x₀=10 出發，η=0.1：

```
Step 0: x₀=10.000, f=100.000, ∇f=20.000, x₁=10-0.1×20=8.0
Step 1: x₁=8.000,  f=64.000,  ∇f=16.000, x₂=8-0.1×16=6.4
Step 2: x₂=6.400,  f=40.960,  ∇f=12.800, x₃=6.4-0.1×12.8=5.12
Step 3: x₃=5.120,  f=26.214,  ∇f=10.240, x₄=5.12-0.1×10.24=4.096
...
Step 10: x₁₀=1.074, f=1.153,  ∇f=2.147
Step 20: x₂₀=0.122, f=0.015,  ∇f=0.243
Step 30: x₃₀=0.014, f=0.0002, ∇f=0.028
```

經過 30 步，x 從 10 降至 0.014，loss 從 100 降至 2×10⁻⁴。注意 |∇f| 隨 x 減小而自動縮小，這是梯度下降對二次函數的「自然退火」。誤差 |xₜ−0| 每步乘以 (1−2η)，η=0.1 時乘 0.8，呈指數衰減。

## 三種變體的實際梯度計算範例

假設 4 個樣本的簡單資料集，每個樣本的損失：

```
L₁(θ)=(θ-1)² → ∇L₁(θ)=2(θ-1)
L₂(θ)=(θ+1)² → ∇L₂(θ)=2(θ+1)
L₃(θ)=(θ-2)² → ∇L₃(θ)=2(θ-2)
L₄(θ)=(θ+2)² → ∇L₄(θ)=2(θ+2)
```

在 θ=0 處計算梯度：

| 方法 | 梯度計算 | 結果 |
|------|---------|------|
| Batch GD | (1/4)Σᵢ∇Lᵢ(0) | (1/4)(-2+2-4+4)=0 |
| SGD (樣本 1) | ∇L₁(0)=2(0-1) | -2 |
| SGD (樣本 2) | ∇L₂(0)=2(0+1) | 2 |
| Mini-batch B=2 (1,3) | (1/2)(∇L₁+∇L₃) | (1/2)(-2-4)=-3 |
| Mini-batch B=2 (2,4) | (1/2)(∇L₂+∇L₄) | (1/2)(2+4)=3 |

Batch GD 給出精確梯度 0（已在極值點），SGD 在 -2 與 2 間搖擺，mini-batch 則介於兩者。這直觀展現了三者的雜訊特性與 batch size 的折衷。

## 學習率對二次函數的影響

用 f(x)=x², x₀=5 比較三種學習率：

### η=0.1（適當）
```
x₀=5.000 → x₁=4.000 → x₂=3.200 → x₃=2.560 → ...
```
穩定收斂，誤差每步乘 0.8。

### η=1.0（邊界震盪）
```
x₀=5.000 → x₁=-5.000 → x₂=5.000 → x₃=-5.000
```
x₁=x₀−1.0×2x₀=−x₀，精準在 ±5 間往返，不收斂亦不發散。

### η=10.0（發散）
```
x₀=5.000, ∇f=10  → x₁=5-10×10=-95
x₁=-95,   ∇f=-190 → x₂=-95-10×(-190)=1805
x₂=1805,  ∇f=3610 → x₃=1805-10×3610=-34295
```
參數與損失每一步都在爆炸。

對於 f(x)=x²，穩定條件 |1−2η|<1 即 0<η<1。η=1 在邊界震盪，η>1 必然發散。這說明學習率的上限由損失函數的曲率（Hessian 的最大特徵值）決定。

## 與牛頓法的比較

牛頓法使用二階資訊：

```
θ_{t+1} = θ_t − H⁻¹ ∇L(θ_t)
```

### 對 f(x)=x² 的比較

| 方法 | 更新 | 從 x₀=10 出發 |
|------|------|--------------|
| 梯度下降 η=0.1 | x←x−0.2x | 約 30 步到 x≈0.014 |
| 牛頓法 | x←x−(2x)/2=0 | **一步收斂** |

牛頓法對純二次函數一步收斂，因為二階資訊精確描述了曲率。

### 為什麼 LLM 不能用二階方法

LLM 參數 N=7B 時：

- **記憶體**：Hessian 是 N×N ≈ 5×10¹⁹ 個元素，約 2×10¹⁹ bytes，遠超所有硬體總和
- **計算**：求逆 O(N³) ∼ 10²⁹ 次操作，完全不可行
- **近似方法**：L-BFGS、K-FAC 等擬牛頓法雖降低到 O(N)，但在雜訊環境下不穩定，且無法充分利用 GPU 批次平行計算

深度學習中一階方法（SGD/AdamW）佔絕對主導地位。

## 自動微分與計算圖

PyTorch 的 `autograd` 是梯度下降能實用的關鍵基礎設施。

### 計算圖的建立

```python
x = torch.tensor([3.0], requires_grad=True)
y = x ** 2        # 前向：計算 y=9
z = y.mean()      # loss 為純量
```

這段程式建立一個有向無環圖：

```
x(requires_grad=True)
    ↓
Pow(x, 2)  ← 保存 x=3 供反向使用
    ↓
Mean       ← 保存 y=9
    ↓
z(loss)
```

### 反向傳播過程

```python
z.backward()  # print(x.grad) → 6.0
```

觸發鏈式法則的遞迴：

1. `dz/dy` = 1（Mean 的梯度）
2. `dy/dx` = 2x = 6（Pow 的梯度）
3. `dz/dx` = 1 × 6 = 6（鏈式法則）

### 動態計算圖的特性

PyTorch 使用 Define-by-Run 模式：每次前向傳播都**重新建立**計算圖，backward 後預設釋放中間節點。這使得模型結構可以動態變化（如不同長度的序列、條件分支）。對比之下，TensorFlow 1.x 的靜態圖（Define-and-Run）需要在執行前先構建完整的圖結構。

## 高維空間中的局部極小值與鞍點

### 鞍點比局部極小值更嚴重

對於高維非凸最佳化，**鞍點（Saddle Point）**遠比局部極小值常見。

範例：`f(x,y)=x²−y²`，在 (0,0) 處 ∇f=0，但 Hessian 特徵值為 +2（x 方向凹向上）與 −2（y 方向凹向下），是一個鞍點而非極小值。

### 機率論解釋

在 d 維空間中，一個臨界點（∇f=0）的 Hessian 有 k 個負特徵值的機率約為 2⁻ᵈ·C(d,k)。當 d 很大時：

| 類型 | 條件 | 機率 |
|------|------|------|
| 局部極小值 | 所有特徵值 > 0 | ≈ 2⁻ᵈ，指數級罕見 |
| 局部極大值 | 所有特徵值 < 0 | ≈ 2⁻ᵈ，同 |
| 鞍點 | 混合正負 | ≈ 1−2⁻ᵈ⁺¹，幾乎必然 |

對於 d=1000 的網路，找到真正局部極小值的機率約為 2⁻¹⁰⁰⁰，可以忽略。所有深度網路的「收斂點」都是鞍點或非常平坦的平台。

### SGD 的雜訊有助於逃離鞍點

在鞍點附近梯度接近 0，純 GD 會停滯。SGD 的梯度雜訊提供隨機擾動，幫助模型沿負曲率方向逃離。

## 樂透彩券假說

Frankle & Carbin (2019) 的樂透彩券假說（Lottery Ticket Hypothesis）：

> 一個隨機初始化的密集神經網路包含一個子網路，該子網路單獨訓練可達到與原始網路相當的測試準確率。

### 與梯度下降的關係

1. **初始化至關重要**：找到的「贏家彩券」子網路必須**保留原始初始化權重**才能成功訓練。重新初始化則訓練失敗。
2. **訓練的雙重任務**：梯度下降同時在完成結構搜尋（哪些連接重要）和參數最佳化（調整權重值）。
3. **剪枝迭代**：訓練→剪枝→ rewinding 回初始權重→再訓練，反覆迭代找到越來越小的子網路。

### 對 mini-llm 的啟示

這解釋了為什麼學習率和初始化範圍如此敏感：它們決定了梯度下降能否發現並訓練一組好的「彩券子網路」。

## 全批次 GD 與 SGD 的收斂路徑比較

### 收斂行為對比

| 特性 | Batch GD（確定性） | SGD（隨機性） |
|------|------------------|--------------|
| 梯度方向 | 每次相同，精確 | 每次不同，有雜訊 |
| 收斂曲線 | 單調下降 | 波動中下降 |
| 極小值類型 | 停在最近的極小值 | 傾向落入寬廣極小值 |
| 泛化能力 | 可能過擬合 | 通常泛化更好 |
| 逃離鞍點 | 困難 | 雜訊輔助逃離 |

### 幾何直覺

在損失等高線圖上：
- **Batch GD** 沿最陡方向直線前進，路徑平滑，但容易陷入淺的窄谷
- **SGD** 路徑曲折隨機，但雜訊使其能越過淺障礙，最終落在寬廣的谷底

經驗上，SGD 找到的極小值 Hessian 特徵值較小（更「平坦」），對應更好的泛化能力。Mini-batch GD (B=32~512) 居於兩者之間，是實務上的最佳折衷。

## 梯度檢查

梯度檢查（Gradient Checking）用數值方法驗證反向傳播是否正確。

### 數值梯度公式

中心差分（誤差 O(h²)，優於單側差分的 O(h)）：

```
f'(x) ≈ [f(x+h) − f(x−h)] / (2h)
```

h 通常取 10⁻⁵。

### 實作範例

```python
def gradient_check(model, x, y, h=1e-5):
    analytical = {n: p.grad.clone() for n, p in model.named_parameters()}
    for name, p in model.named_parameters():
        numerical = torch.zeros_like(p)
        flat = p.view(-1)
        for i in range(flat.shape[0]):
            orig = flat[i].item()
            flat[i] = orig + h
            loss_p = model(x, y)
            flat[i] = orig - h
            loss_m = model(x, y)
            flat[i] = orig
            numerical.view(-1)[i] = (loss_p - loss_m) / (2 * h)
        err = (analytical[name] - numerical).norm() / \
              (analytical[name].norm() + numerical.norm() + 1e-12)
        print(f"{name}: rel_err = {err:.2e}")
```

### 判斷標準

| 相對誤差 | 含義 |
|---------|------|
| < 10⁻⁷ | 完美 |
| 10⁻⁷ ~ 10⁻⁵ | 良好 |
| 10⁻⁵ ~ 10⁻³ | 可能需檢查 |
| > 10⁻³ | 反向傳播很可能有錯誤 |

注意：梯度檢查非常慢（每個元素需兩次前向傳播），僅適用於小模型抽查。對 LLM 只能抽查少量參數。檢查時需關閉 Dropout/BatchNorm 等隨機層。

## 連接 mini-llm：單步訓練完整追蹤

以下是 mini-llm 中一個完整的訓練步驟：

```python
# 1. 前向傳播 — 計算 logits 與 loss
logits = model(x)                                              # [B, seq_len, vocab_size]
loss = F.cross_entropy(logits.view(-1, vocab_size), y.view(-1))

# 2. 反向傳播 — 自動微分計算所有參數梯度
loss.backward()                                                # 填充 param.grad

# 3. 梯度裁切 — 防止梯度爆炸
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 4. 參數更新 — AdamW
optimizer.step()                                               # θ ← θ − η·adamw(θ, grad)

# 5. 清空梯度 — 為下一步做準備
optimizer.zero_grad()
```

### 各步驟的張量與內存

假設 B=32, seq_len=64, vocab_size=65, d_model=128：

| 步驟 | 關鍵張量 | 維度 | 說明 |
|------|---------|------|------|
| forward | logits | [32,64,65] | 模型輸出 |
| loss | 純量 | [1] | 交叉熵 |
| backward | grads | 與參數同形狀 | 每個參數對應的梯度 |
| clip | 縮放後的 grads | 同 grad | 確保 ‖grad‖ ≤ 1.0 |
| step | 更新後的參數 | 同參數 | θ ← θ − η·mₜ/(√vₜ+ε) |

### 內存尖峰

在 `loss.backward()` 執行時，PyTorch 需要保留所有中間啟用值。這意味著：

- **前向階段**：保存每個 Transformer block 的輸出、attention scores 等
- **反向階段**：從 loss 開始逐層計算梯度，並逐步釋放中間變數
- **訓練總記憶體** ≈ 參數 × 3（參數+梯度+優化器狀態）+ 啟用值記憶體

對 7B 模型，AdamW 優化器狀態約需 7B × 4 bytes × 3 ≈ 84 GB，這解釋了為什麼大模型訓練需要分散式系統和記憶體節省技術（如梯度 checkpointing、ZeRO 等）。
