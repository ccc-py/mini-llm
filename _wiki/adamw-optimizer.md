# AdamW Optimizer — AdamW 最佳化器

## 概述

**AdamW** 是一種流行的深度學習最佳化演算法，由 Loshchilov 與 Hutter 在 2019 年的論文「Decoupled Weight Decay Regularization」中提出。AdamW 是 Adam 最佳化器的改良版本，關鍵改進是將**權重衰減（Weight Decay）** 與**自適應學習率**的更新步驟解耦，以解決 Adam 中的 L2 正則化與自適應學習率之間的不良互動。mini-llm 所有版本的訓練都使用 AdamW 作為最佳化器。

## 動機：從 SGD 到 Adam

### 隨機梯度下降（SGD）

SGD 是最基本的更新規則：

```
θ_{t+1} = θ_t - η · ∇L(θ_t)
```

其中 `η` 是學習率，`∇L(θ_t)` 是損失對參數的梯度。

SGD 的問題：
- 對所有參數使用相同的學習率
- 在梯度方向變化大的區域（如 ravine）容易震盪
- 需要仔細調整學習率排程

### 動量（Momentum）— SGD 的改進

加入動量項來平滑更新方向：

```
v_{t+1} = β₁·v_t + (1-β₁)·∇L(θ_t)
θ_{t+1} = θ_t - η·v_{t+1}
```

動量讓更新方向累積過去梯度的加權平均，在梯度變換方向時減少震盪。

### RMSProp — 自適應學習率

RMSProp 為每個參數維護不同的學習率：

```
v_t = β₂·v_{t-1} + (1-β₂)·(∇L(θ_t))²
θ_{t+1} = θ_t - η·∇L(θ_t) / (√v_t + ε)
```

頻繁更新的參數獲得較小的學習率，稀少更新的參數獲得較大的學習率。

### Adam — 動量 + 自適應學習率

Adam（Kingma & Ba, 2014）結合了動量和 RMSProp 的優點：

```
m_t = β₁·m_{t-1} + (1-β₁)·g_t                    ← 梯度的一階動量（均值）
v_t = β₂·v_{t-1} + (1-β₂)·g_t²                    ← 梯度的二階動量（未中心化方差）
m̂_t = m_t / (1 - β₁^t)                              ← 偏差校正
v̂_t = v_t / (1 - β₂^t)
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε)              ← 更新
```

其中：
- `β₁` — 一階動量的衰減率（通常 0.9）
- `β₂` — 二階動量的衰減率（通常 0.999）
- `ε` — 數值穩定常數（通常 1e-8）

Adam 是深度學習中最成功的最佳化器之一，尤其適合：
- 大規模參數
- 稀疏梯度
- 非平穩目標
- 嘈雜的梯度

## Adam 的問題：L2 正則化與自適應學習率

### L2 正則化（權重衰減）

L2 正則化在損失函數上添加權重平方和的懲罰項：

```
L'(θ) = L(θ) + (λ/2) · ||θ||²
```

更新規則變為：

```
θ_{t+1} = θ_t - η · (∇L(θ_t) + λ·θ_t)
         = (1 - η·λ)·θ_t - η·∇L(θ_t)               ← 權重衰減 + 梯度更新
```

### L2 正則化在 Adam 中的問題

當 L2 正則化整合進 Adam 時：

```
g_t = ∇L(θ_t) + λ·θ_t                              ← 加入 L2 懲罰
m_t = β₁·m_{t-1} + (1-β₁)·g_t
v_t = β₂·v_{t-1} + (1-β₂)·g_t²
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε)
```

問題在於：L2 懲罰項 `λ·θ_t` 被 Adam 的自適應學習率機制同時縮放。對於梯度稀疏的參數，`v_t` 小，`1/(√v̂_t + ε)` 大，導致 L2 懲罰被放大；對於梯度密集的參數則相反。這導致權重衰減的效果不再均勻，正則化強度因參數而異，難以調控。

## AdamW 的解決方案

AdamW 的改進極其簡單：**將權重衰減從梯度計算中移除，直接在參數更新時應用**。

### AdamW 更新規則

```
m_t = β₁·m_{t-1} + (1-β₁)·∇L(θ_t)                 ← 原始梯度（不含 L2 項）
v_t = β₂·v_{t-1} + (1-β₂)·(∇L(θ_t))²
m̂_t = m_t / (1 - β₁^t)
v̂_t = v_t / (1 - β₂^t)

θ_{t+1} = θ_t - η · (m̂_t / (√v̂_t + ε) + λ·θ_t)   ← 解耦的權重衰減
```

權重衰減 `λ·θ_t` 在自適應學習率之外獨立應用，不受 `√v̂_t` 的影響。這樣：
- 正則化強度均勻 — 所有參數的權重衰減率一致
- 控制簡單 — `λ` 的意義清晰，容易調參
- 效果更好 — 在許多基準上優於 Adam 的 L2 正則化

### mini-llm 的實作

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=learning_rate)
```

PyTorch 的 `torch.optim.AdamW` 已經整合了「解耦的權重衰減」。相關參數：

```python
# PyTorch AdamW 的預設參數
torch.optim.AdamW(
    params,
    lr=0.001,               # 學習率
    betas=(0.9, 0.999),     # (β₁, β₂) — 一階與二階動量衰減率
    eps=1e-8,                # 數值穩定常數
    weight_decay=0.01,       # 權重衰減係數 λ
    amsgrad=False            # AMSGrad 變體
)
```

mini-llm 使用 `lr=3e-4`（v1）或 `lr=5e-4`（v2/v3 pretrain）、`lr=1e-4`（v2/v3 finetune），其餘保持預設。

## AdamW 的超參數

| 參數 | 典型值 | 對訓練的影響 |
|------|--------|-------------|
| `learning_rate` | 1e-5 ~ 5e-4 | 最重要的超參數。太大不收斂，太小收斂慢 |
| `β₁` | 0.9 | 控制一階動量的衰減。越大代表考慮更多歷史梯度 |
| `β₂` | 0.999 | 控制二階動量的衰減。對稀疏梯度任務敏感 |
| `ε` | 1e-8 | 防止除以零的常數。通常不需要調整 |
| `weight_decay` | 0.01 ~ 0.1 | 控制正則化強度。越大權重衰減越強 |

## SGD vs Adam vs AdamW

| 特性 | SGD + Momentum | Adam | AdamW |
|------|----------------|------|-------|
| 自適應學習率 | 無 | 是 | 是 |
| 權重衰減方式 | 內建於 L2 正則化 | L2 正則化（不自適應） | 解耦（獨立應用） |
| 收斂速度 | 慢 | 快 | 快 |
| 泛化能力 | 好（經驗上） | 有時不如 SGD | 與 SGD 相當或更好 |
| 超參數敏感度 | 較高（學習率關鍵） | 較低 | 較低 |
| 記憶體用量 | 低（1 個狀態/參數） | 高（2 個狀態/參數） | 高（2 個狀態/參數） |

## Brownian Bridge 的類比

Adam 的更新規則可以直觀理解為：在參數空間中，每次更新嘗試朝著（一階動量指示的）下坡方向前進，步長由（二階動量指示的）梯度方差來調整 — 梯度方差大的方向步長縮小，梯度方差小的方向步長放大。AdamW 在此基礎上，對所有參數施加獨立的指數衰減（權重衰減），作為一種持續的正則化。

## 具體更新範例

假設 `θ=1.0`，`g=0.1`，`η=1e-3`，`λ=0.01`，`β₁=0.9`，`β₂=0.999`。

**t=1**（初始 `m₀=0`，`v₀=0`）：

```
m₁ = 0.9·0 + 0.1·0.1 = 0.01
v₁ = 0.999·0 + 0.001·0.01 = 0.00001
m̂₁ = 0.01 / (1 - 0.9) = 0.1
v̂₁ = 0.00001 / (1 - 0.999) = 0.01
θ₁ = 1.0 - 0.001·(0.1/√0.01 + 0.01·1.0) = 0.99899
```

**t=2**（梯度仍為 0.1）：

```
m₂ = 0.9·0.01 + 0.1·0.1 = 0.019
v₂ = 0.999·0.00001 + 0.001·0.01 ≈ 0.00002
θ₂ = 0.99899 - 0.001·(0.1/0.1 + 0.01·0.999) = 0.99798
```

觀察：**即使梯度不變，權重衰減也持續將參數推向零**。每步的衰減量約 `η·λ·θ ≈ 1e-5·θ`，與參數大小成正比。這是正則化的核心作用 — 參數越大，受到的衰減拉力越強，防止過度增長。

## 二維損失曲面的直觀比較

想像一個橢圓形損失等高線（長軸平坦、短軸陡峭）。

### SGD

更新方向垂直於等高線。在陡峭方向反覆震盪，平坦方向前進極慢。路徑呈「之」字形，收斂效率低。

### SGD + Momentum（β₁=0.9）

動量累積過去梯度的加權平均。陡峭方向的正負梯度相互抵消、震盪衰減；平坦方向的同向動量持續累積、有效加速。路徑為平滑螺旋線。

### Adam（β₁=0.9, β₂=0.999）

自適應學習率：陡峭方向（大梯度→大 v→小步長）步長縮小，平坦方向（小梯度→小 v→大步長）步長放大。相當於將橢圓等高線「圓形化」，各方向更新步長趨於一致。收斂極快。

### AdamW

路徑與 Adam 相同，但多了持續的徑向收縮力 — 參數向量被穩定拉回原點。長時間訓練下參數範數小於 Adam，有助於提升泛化。

## 偏差校正分析

為什麼需要 `m̂_t = m_t / (1 - β₁^t)`？

初始 `m₀=0`，t=1 時 `m₁ = (1-β₁)·g₁ = 0.1·g₁` — 只有真實梯度的 1/10！若直接用 m₁ 更新，第 1 步步長會嚴重偏小。經校正 `m̂₁ = 0.1·g₁/(1-0.9) = g₁`，恢復正確大小。

t → ∞ 時 β₁^t → 0，校正因子 → 1。t=50 時 0.9⁵⁰≈0.005，影響 < 0.5%。t=500 時校正因子 ≈ 1.00000003，可忽略。

二階動量同理：t=1 時 `v₁ = (1-β₂)·g₁² = 0.001·g₁²`，校後 `v̂₁ = g₁²`。因 β₂=0.999，t=1 時偏差達 1000×，t=1000 時 β₂¹⁰⁰⁰≈0.368，校正因子約 1/(1-0.368)≈1.58，仍顯著。

## 權重衰減的直觀理解

權重衰減可以看作對參數向量施加指數衰減：`θ ← (1 - η·λ)·θ`。每一步都將參數向原點縮小一點點。

若不使用權重衰減，參數範數在訓練過程中會持續增長（特別是那些梯度一致的方向），導致模型對訓練資料過度擬合。權重衰減的作用就是遏制這種增長，類似物理中摩擦力對物體運動的持續阻礙。

## 權重衰減對 0.8M 參數的實際影響（mini-llm v1）

每步衰減 `Δθ = -η·λ·θ`。`η=3e-4`，`λ=0.01` → 每步衰減 `3e-6·θ`。2000 步後累計 `(1-η·λ)²⁰⁰⁰ ≈ exp(-0.006) ≈ 0.994`，參數範數縮減約 0.6%。

分層影響：嵌入層（~300K，範數 ~1-2）衰減較明顯；FFN 層（~400K，範數 ~0.1-0.5）較弱。在 0.8M 規模下，weight_decay=0.01 主要防止參數後期過度增長。若擴大到 100M+ 參數，高維空間中範數自然更大，同樣 λ 效果更強。

## 超參數敏感度分析

| 超參數 | 敏感度 | 說明 |
|--------|--------|------|
| `lr` | ★★★★★ | 偏離 3 倍即不收斂或收斂極慢 |
| `β₁` | ★★☆☆☆ | 0.8~0.95 穩定 |
| `β₂` | ★★☆☆☆ | 0.99~0.9999 穩定，稀疏梯度任務敏感 |
| `weight_decay` | ★★★☆☆ | 小模型 0~0.1 均可；大模型 (>100M) 高度敏感 |
| `ε` | ★☆☆☆☆ | 1e-6~1e-10 幾乎無影響 |

小模型 (<10M) LR 範圍寬鬆（1e-4~1e-3）；大模型 (>1B) 收窄至 1e-4~3e-4。

## PyTorch AdamW vs 原始論文

兩者在數學上完全等價。原始論文公式：

```
θ_{t+1} = θ_t - η·m̂_t/(√v̂_t + ε) - η·λ·θ_t
```

PyTorch 實作將偏差校正融入 step_size，並對公式進行代數重排：

```python
step_size = lr / (1 - β₁^t)
bias_correction₂ = √(1 - β₂^t)
p *= (1 - lr·weight_decay)     # 解耦衰減
p -= step_size · m / (√v / bias_correction₂ + ε)
```

細微差異：論文的 ε 在 `√v̂ + ε` 中，PyTorch 的 ε 在 `√v/bias_correction₂ + ε` — 但代數上完全等價。PyTorch 額外支援 AMSGrad 變體（預設關閉）。**可以直接信賴 `torch.optim.AdamW`**。

## 記憶體開銷

AdamW 每參數存 `m`（4B）+ `v`（4B）+ 參數本身（4B）= 12B = 3× 模型大小。

| 模型 | 參數量 | 模型 | AdamW 狀態 | 總計 |
|------|--------|------|-----------|------|
| v1 | 0.8M | 3.1 MB | 6.2 MB | 9.3 MB |
| 7B LLM | 7B | 28 GB (bf16) | 56 GB (fp32) | 84 GB |

mini-llm 無記憶體壓力；7B 模型超過單張 A100 (80GB)。大規模訓練常用 8-bit AdamW、Adafactor 或 SGD+momentum。

## 替代方案比較

| 最佳化器 | 記憶體 | 收斂速度 | 主要優勢 | 主要劣勢 |
|---------|--------|---------|---------|---------|
| **AdamW** | 3× | 快 | 通用性最強，幾乎無需調參 | 記憶體開銷大 |
| **SGD+Momentum** | 1× | 中 | 記憶體少，泛化好 | 需仔細調 LR 和排程 |
| **Sophia** | 2× | 極快 | 基於 Hessian 估計，收斂快 2× | 實現複雜，對超參數敏感 |
| **Shampoo** | 4-9× | 快 | 考慮梯度協方差，各向異性步長 | 計算 O(d²)，記憶體極大 |
| **Adafactor** | 1× | 中 | 分解 v 為行/列和，記憶體極省 | 預設參數未必通用 |
| **LION** | 1× | 快 | 無動量狀態，僅追蹤 sign | 需特殊初始化和 LR |

選用建議：通用首選 AdamW；記憶體受限用 Adafactor 或 8-bit AdamW；追求泛化用 SGD+Momentum（CV 常見）；追求速度用 Sophia（LLM 預訓練）。mini-llm 場景下 AdamW 完全足夠，無需更換。

## mini-llm 學習率選擇原理

- **v1 pretrain (3e-4)**：3 層 Transformer、0.8M 參數、2000 步從頭訓練。3e-4 是小模型的甜蜜點：夠大快速收斂、夠小不會發散，搭配餘弦退火排程
- **v2/v3 pretrain (5e-4)**：4 層 1.1M 參數。稍高 LR 是因為網絡更深（梯度信噪比更高）、訓練步數僅 500 步需要更快收斂
- **v2/v3 finetune (1e-4)**：從預訓練權重微調。LR 為預訓練的 1/5，防止破壞已學知識（災難性遺忘）、適應少量微調資料。這也是 LLM 微調的常見模式：pretrain 用較高 LR 快速學習通用表徵，finetune 用較低 LR 微調特定任務

經驗法則：微調 LR ≈ 預訓練 LR × 0.1~0.3。

## 何時從 AdamW 切換到 SGD

- **大規模微調 (>1B)**：不建議切換，風險高且收益不明確
- **小資料集精調**：先用 AdamW 快速收斂，後切換 SGD+momentum 尋找更平坦的極小值，提升泛化
- **記憶體瓶頸**：AdamW 需 3× 記憶體，SGD 只需 1×，可換取更大 batch size
- **mini-llm**（<2M 參數）：全程 AdamW 即可，無需切換
